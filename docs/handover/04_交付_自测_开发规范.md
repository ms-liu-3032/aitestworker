# 交付、自测与开发规范

## 1. 交付标准

每次交付至少满足：

1. 能说明这次改了什么
2. 能说明影响哪些模块
3. 能说明如何启动和验证
4. 构建通过
5. 不破坏现有主链路

## 2. 自测标准

### 前端

必须执行：

```bash
cd <repo-root>/frontend
npm run build
npx vue-tsc --noEmit
```

### 后端

必须执行：

```bash
cd <repo-root>/backend
mvn test
```

### 本地 worker

必须执行：

```bash
cd <repo-root>/local-browser-worker
npm run build
npm test
```

### 安装包

每次改打包脚本后至少验证：

1. 打包脚本能成功生成产物
2. 安装脚本可执行
3. 安装说明与脚本行为一致

## 3. 开发规范

1. 中文注释优先，简洁明确。
2. 不引入自由 Agent 逻辑。
3. 所有 LLM 调用都走统一配置。
4. 轨迹生成的是功能测试用例，不是自动化脚本。
5. 原始轨迹事实不可被 AI 覆盖，只能在清洗层和语义层上加工。

## 4. 变更记录规范

从现在开始，每次改动必须同时更新：

1. [CHANGELOG.md](<repo-root>/docs/handover/CHANGELOG.md)
2. [05_上下文日志.md](<repo-root>/docs/handover/05_上下文日志.md)

### 最小记录模板

```text
日期：
范围：
变更：
原因：
影响：
验证：
```

### 记录要求

1. 不要写“优化了一些内容”这种空话。
2. 必须能看出是哪个模块、哪条链路、怎么验证的。
3. 一次交付一条，不合并历史事实。

## 5. 交接给新 token / 新模型的最低要求

新接手方必须先看：

1. [README.md](<repo-root>/docs/handover/README.md)
2. [05_上下文日志.md](<repo-root>/docs/handover/05_上下文日志.md)
3. [CHANGELOG.md](<repo-root>/docs/handover/CHANGELOG.md)

并在开始修改前确认：

1. 当前优先级
2. 当前可运行方式
3. 当前未完成项
4. 当前自测命令
