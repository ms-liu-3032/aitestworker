# 项目与设计总览

## 1. 项目名称

智能测试平台

## 2. 项目定位

这是一个面向企业测试工作的受控智能平台，不做自由 Agent，不做完全模板流，而是：

1. 用结构化流程承接测试工作。
2. 在固定节点调用受控 LLM。
3. 把测试过程中的知识、轨迹、用例、Skill、Tool 沉淀成资产。

## 3. 当前范围

当前实现聚焦第一阶段：

1. 主平台
   - 登录、用户、项目、模型配置、公共提示词库
   - 测试用例生成
   - 测试执行轨迹采集器
   - 用户本地用例库 / 正式用例库
   - 测试资产沉淀
   - 管理后台
2. 本地客户端
   - `local-browser-worker`
   - 本地浏览器打开
   - 基础操作事件采集
   - 基础网络请求摘要采集
   - 绑定主平台
3. 受控扫描能力
   - 页面画像扫描
   - 项目级扫描结果参与轨迹清洗和用例生成

## 4. 当前不做的事

第一阶段明确不做：

1. 自由 Agent 自主决策系统
2. 桌面级录屏
3. 自动回放
4. 自动提缺陷单
5. 自动绕过验证码 / MFA
6. Edge 支持
7. 桌面客户端安装器正式发布版（DMG/MSI 级别）

## 5. 当前产品主链路

### 5.1 测试用例生成

1. 用户输入需求
2. 选择模型与提示词
3. 创建生成任务
4. 直连模型生成草稿
5. 草稿进入本地库
6. 后续确认并提交正式库

### 5.2 测试执行轨迹

1. 创建身份空间
2. 绑定本地 worker
3. 打开浏览器并开始采集
4. 采集事件、网络、提示消息
5. 停止后写回主平台
6. 清洗轨迹
7. 生成摘要 / 用例 / Skill / Tool

### 5.3 页面理解与受控扫描

1. 扫描可达页面
2. 建页面画像
3. 页面画像参与轨迹清洗
4. 页面画像参与功能测试用例生成

## 6. 关键设计原则

1. 原始事实保留，AI 只做受控整理。
2. 轨迹生成的是功能测试用例，不是自动化脚本。
3. 页面适配靠通用模式 + 项目级画像，不靠逐页人工硬编码。
4. 任何高风险动作都要可追踪、可回溯。

## 7. 开源平台定位（2026-06-13 更新）

平台默认启动时为纯通用内核，不绑定任何特定业务。业务能力通过项目使用过程中自动沉淀的 business pack 持续演化。

### 架构硬规则

1. 业务差异不得优先写进 Java 代码。
2. 平台核心只保留通用流程、通用规则引擎、通用扫描机制、通用语义沉淀机制。
3. 业务特征只能存在于数据库、JSON 配置、自动沉淀生成的 business pack。
4. 历史样例资产（如 `trace-rulepacks-samples/`）不参与默认运行态。
5. 新业务接入不需要新增 Java 业务包类。
6. 已发布迁移脚本不得为了清理文案而改写；默认值和注释修正通过后续迁移收口。

### business pack 自动沉淀

系统根据项目资产（页面扫描、轨迹录制、TOM、摘要等）自动生成 business pack draft，支持人工确认、合并、拆分、启用/停用/归档。

### business pack 数据模型

```
business_pack (主表)
  ├── business_pack_item (条目：TOM/PAGE/FIELD/ACTION/FLOW/STATE/ASSERTION/TERM)
  ├── business_pack_relation (包间关系：DEPENDS_ON/CONTAINS/LINKED/SUPPLEMENTS)
  ├── business_pack_snapshot (版本快照：每次状态变更自动记录)
  ├── business_pack_rule_binding (规则绑定：与轨迹清洗/描述规则关联)
  ├── business_pack_scan_binding (扫描绑定：与页面画像关联)
  ├── business_pack_tom_binding (TOM 绑定：与测试对象模型关联)
  ├── business_pack_semantic_binding (语义绑定：与语义包关联)
  ├── business_pack_consumption_log (消费记录：追踪哪些链路消费了业务包)
  └── business_pack_refresh_diagnostic (自动刷新诊断：成功/失败、输入规模、生成数量)
```

### 规则包与扫描源配置化

- `trace_rule_pack_config` 用于数据库化轨迹规则包，当前按项目加载项目级 ACTIVE 规则包，并叠加全局 ACTIVE 规则包。
- 项目规则包前端提供输入字段、选择项、点击动作、工作流骨架模板，并在保存前做基础结构检查和规则预览。
- `scan_source_config` 用于全局 / 项目级扫描源配置；项目页只允许修改当前项目扫描源，全局扫描源在项目页只读展示，运行时只加载启用源。
- `BuiltinScanSourceProvider` 与 classpath `trace-rulepacks/*.json` 均保留为兼容扩展，不作为新业务默认接入方式。

### 前端产品化

- 路由页面已使用 `React.lazy` + `Suspense` 按需加载，默认构建不再产生 Vite 大 chunk warning。

### business pack 生命周期

```
DRAFT → ACTIVE → INACTIVE → ARCHIVED
  ↓       ↓         ↓
ARCHIVED ARCHIVED  ARCHIVED
```

### business pack 来源追溯

每个条目和绑定都可追溯到具体项目资产：
- 条目来源：TOM / PAGE_SCAN / TRACE_CORRECTION / TRACE_SUMMARY
- 规则绑定：trace rule / cleaning pattern / description rule
- 扫描绑定：page_scan_profile
- TOM 绑定：test_object_model
- 语义绑定：semantic_pack
- 消费记录：TRACE_CLEAN / SCOPE_ANALYSIS / CASE_GENERATION / SUMMARY / SEMANTIC_CONTEXT
- 自动刷新诊断：记录 TOM、页面画像、步骤模板、摘要输入数量，以及生成包、自动推断关系和失败原因
